# Source Generators

This page is for contributors working on framework source generators.

## Contributor Goals

Source generators should remove repetition while keeping generated behavior traceable to explicit source declarations.

## Documentation Expectations

When adding or changing a generator, update documentation in the relevant places:

- Concepts for the mental model.
- Guides for consumer usage.
- Reference for directives, attributes, generated shapes, or configuration.
- Decisions if the change introduces a new architectural trade-off.

## Consumer Boundary

Consumer documentation should explain what the generator does and what declarations trigger it. Contributor documentation can explain implementation details.
